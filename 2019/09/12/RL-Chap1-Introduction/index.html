REINFORCEMENT LEARNING: AN INTRODUCTION

Richard S. Sutton and Andrew G. Barto

Chapter 1 Introduction

1. Main Assumptions
  • Agent can sense the state of its environment;
  • Agent can take actions that affect the state;
  • Agent has goals relating to the state of the environment.

2. Main Features

  • Trial-and-error search

    最初的学习器是不知如何做决策的，通过试错来学习。

  • Delayed reward

    未来的，隐性的奖励，但在目标中需要考虑的。

  • Trade-off between exploration and exploitation

    • exploration：类似最速梯度下降不选择梯度负方向，避免落入局部最小值；
    • exploitation：是一个greedy的过程，每步最大化。
3. Unsupervised learning v.s Reinforcement learning
  • RL是无标签的，需要从环境反馈的reward猜测之前采取的那一步是好是坏；
  • Unsupervised Learning 是寻找无标签样本间的关系；
  • RL 是有一个明确目标函数的，有时即使我们找到了样本间的关系，完全有可能仍无法使目标最大；
  • RL 与时间t密切相关。
4. Main Elements
Policy (main)

  • 离散，有限情形(本书第一部分)

  • 连续情形(本书第二部分)

    $f:$ {State} $\to$ {Action}​

    $f:$ {State} $\times$ {Reward} $\to$ {Action}

Reward signal
  • Given directly by the environment;
  • $h:$ {State} $\times$ {Action} $\to$ {Reward}.
Value function
  • Must be estimated;
  • Total amount of reward in future.

How to estimate？

Environment model
  • Only needed in model-based methods；
  • Model-Free.
5. 以井字棋为例
可选方案

  • minimax

    无法实现伪败设陷阱。

  • dynamic programming

    需要完全了解环境；

    实际问题中，用状态转移概率的无偏估计(和RL类似)。

  • evolutionary method

    生成多个棋手对抗，选择胜率最大的那个棋手；

    生成后棋手不再变化，也即对抗过程中不变。

  • reinforcement learning

    时间差分TD的方法；

    更新Value Function.

模拟退火？

TD learning method
  • 同一情形下，先后手的Value是不同的

  • 初值是一种猜测

    复杂情形中，不必一次性给完，可以一边下一边给。

  • 更新路径

​ $$V(s) \leftarrow V(s)+\alpha\left[V\left(s^{\prime}\right)-V(s)\right]$$

  • 再下一次如果赢率大，那本次“权重”增大。(凸组合)
  • 但这个权重的改变只会在下一轮游戏中起作用，本轮游戏不会再经历这个状态了。不同于Exploration。
Self-play平局的赢率
  • 井字棋存在“完美的棋手”；
  • 先手可将平局赢率设置为1；
  • 后手需要将平局设置为0，否则会永远赢不了先手。
